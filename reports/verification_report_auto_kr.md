### `baseline_definition` ⚠️ **사람 SAP에 내용 있음**
- 프로토콜에 SAP 분석용 기저치(baseline) 정의가 명시되어 있지 않음. 기저치 평가는 Day -1 및 스크리닝에 예정되어 있으나, 각 변수별 기저치 정의(Day -1 마지막 평가, 스크리닝 값, 복수 측정의 평균 등)는 프로토콜에 기재되지 않음.
- **사람 SAP 내용**: Physical Examination(Day 1 투약 전), Vital Signs(Day 1 투약 전), ECG(Day 1 투약 전 평균값), Lab(Day -1), GL1/Gangliosides(Day 1 투약 전) — 변수별 구체적 기저치 정의 제시

### `blinded_analyses` ⚠️ **사람 SAP에 내용 있음**
- 프로토콜에 SRC 보고서 작성 생물통계학자와 최종 비맹검 분석 담당자의 분리 절차가 상세히 기술되어 있지 않음.

### `concomitant_medication` ⚠️ **사람 SAP에 내용 있음**
- WHO Drug Dictionary Enhanced 등 코딩 시스템이 프로토콜에 명시되지 않음
- ATC 분류별, 치료군별 구체적 표 작성 방법이 프로토콜에 상세히 기술되지 않음
- **사람 SAP 내용**: ATC 코드(해부학적 분류 1단계, 2단계)별 선행약물/병용약물 분류 및 집계 방법 구체적 기술

### `derived_variables` ⚠️ **사람 SAP에 내용 있음**
- Phase 1 FIH 시험으로 임상 유효성 평가변수나 파생 유효성 변수가 없음. 프로토콜에 파생 유효성 변수, 기저치 대비 변화량 계산, 반응자 정의 등이 정의되어 있지 않음.

### `fas_definition` ⚠️ **사람 SAP에 내용 있음**
- 프로토콜에 Full Analysis Set(FAS) 정의가 명시되지 않음. 무작위 배정 집단, 안전성 분석군, PK/PD 분석군은 정의되어 있으나, 기저치+기저치 후 측정 요건을 포함한 FAS 정의는 없음.
- **사람 SAP 내용**: "무작위 배정되어 최소 1회 이상 IP를 투여받은 모든 대상자. 최초 배정에 따라 분석" — 관행에 따른 FAS 정의 제시

### `ip_exposure`
- 프로토콜상 대상자 내 용량 조절이 허용되지 않음; 모든 용량 변경은 SRC 결정에 따라 코호트 수준에서 발생

### `lab_char_values` ⚠️ **사람 SAP에 내용 있음**
- 프로토콜에 비수치 임상검사 값(BLQ, ND, '<', '>' 등)의 처리 규칙이 명시되지 않음. 통상 SAP 방법론 섹션 또는 생물분석/임상검사 SOP에 기록됨.

### `medical_history` ⚠️ **사람 SAP에 내용 있음**
- 프로토콜에 SAP용 병력 코딩/분석 절차가 명시되지 않음. 선정/제외 기준에 의학적 조건이 포함되고 스크리닝 시 병력 수집이 확인되지만, MedDRA 코딩 버전, 요약 절차, 분석군 등은 기재되지 않음.
- **사람 SAP 내용**: MedDRA version 27.0 이상, SOC/PT별 병력 빈도/백분율 제시

### `missing_ae_severity` ⚠️ **사람 SAP에 내용 있음**
- 프로토콜에 이상반응 중증도 결측 시 대체 규칙이 명시되지 않음.

### `missing_causality` ⚠️ **사람 SAP에 내용 있음**

### `missing_dates` ⚠️ **사람 SAP에 내용 있음**
- 프로토콜에 이상반응/약물의 불완전한 날짜에 대한 대체 규칙이 명시되지 않음.

### `missing_last_dose_date` ⚠️ **사람 SAP에 내용 있음**
- 프로토콜에 IP 마지막 투여일 결측 시 처리 방법이 구체적으로 제시되지 않음. eCRF/대상자 일지에 투여일시를 기록하도록 요구하나, 결측 시 대체 절차는 기재되지 않음.

### `pps_definition`

- **사람 SAP 내용**: Source data verification 미반영 상태로 분석, 중앙검사실 제공 데이터 사용, 명목 PD 채혈시간 사용, Emax/AUEC 계산 방법, 결과 보고 형식/파일명 규칙까지 상세 기술

## 5. 개선 로드맵

### Phase 1 (현재): 프로토콜 추출 정확도 향상
**목표**: 프로토콜에 있는 내용을 최대한 정확하고 구체적으로 추출

- 프로토콜에 정보가 있는데 [SKIP]하는 경우 감소
- 구체성 향상: 정확한 수식, 버전 번호, 시점 추출
- BLQ 처리 규칙, PK 파라미터, 용량 비례성 상세화

### Phase 2 (다음): 전문가 판단 레이어
**목표**: 프로토콜에 없지만 생물통계 관행상 명확한 내용을 AI가 채움

이것은 **별도 프로젝트**로 진행해야 합니다. 현재 AI SAP은 "프로토콜에 있는 내용을 추출"하는 데 집중하고 있으며, Phase 2는 "프로토콜에 없는 내용을 관행에 따라 생성"하는 근본적으로 다른 접근이 필요합니다.

- 표준 FAS/PPS 정의 생성 (ICH E9 기반)
- 변수별 기저치 정의 생성 (SoA + 관행)
- 결측 데이터 대체 규칙 생성 (업계 표준)
- 프로토콜 일탈 분류 프레임워크 생성
- 통계 소프트웨어 표준 텍스트 생성

**구현 방향**:
1. "관행 라이브러리(Convention Library)" 구축 — 승인된 SAP 예시에서 표준 문구 수집
2. 각 섹션: 프로토콜 추출 시도 → 부족하면 관행 라이브러리에서 보충
3. 관행 기반 내용은 시각적으로 구분 표시 (예: 파란색 텍스트) → 리뷰어가 출처 파악 가능
4. 2단계 생성: Pass 1(프로토콜 추출) → Pass 2(관행 보충)

### Phase 3 (향후): Table Shell 생성
**목표**: 출력 테이블 양식 ~70개 자동 생성 — SAP의 핵심 가치

| 카테고리 | 예상 테이블 수 | 예시 |

## 6. 결론

현재 AI SAP은 **프로토콜에 있는 내용을 추출하는 능력은 양호**합니다 (46개 섹션 중 약 24개가 사람 수준). 그러나 사람 통계학자가 작성하는 SAP와의 핵심적인 차이는 다음 세 가지입니다:

1. **전문가 판단 내용** — 프로토콜에 없어도 관행에 따라 작성하는 정의/규칙 (FAS, 기저치, 결측 처리 등)
2. **구체성** — 정확한 수식, 소프트웨어 버전, 코딩 사전 버전 등
3. **Table Shell** — SAS/R 프로그래머가 바로 사용할 수 있는 출력 테이블 양식

이 세 가지는 각각 Phase 1(구체성), Phase 2(전문가 판단), Phase 3(Table Shell)에서 순차적으로 개선할 계획입니다. **현재 단계에서는 프로토콜 추출 정확도를 최대화하는 데 집중하고**, 복잡한 판단이 필요한 내용은 초안 작성 후 별도 프로젝트로 보완하는 것이 효율적입니다.
